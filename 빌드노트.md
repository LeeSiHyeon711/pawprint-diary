# 빌드노트 — PawprintDiary

## 프로젝트 정보
- 스택: Next.js 14 App Router + React 18 + TypeScript + Tailwind CSS
- 위치: `projects/PawprintDiary/05-개발/` (이 폴더가 Next.js 프로젝트 루트)

---

## 실행 방법

### 1. 의존성 설치
```bash
cd projects/PawprintDiary/05-개발
npm install
```

### 2. 개발 서버 실행
```bash
npm run dev
```
브라우저에서 `http://localhost:3000` 접속

### 3. 프로덕션 빌드 검증
```bash
npm run build
```
타입/빌드 에러 0으로 통과해야 FEAT 완료 조건 충족 (설계서 7장 전역 규칙)

---

## 환경 변수 설정

`.env.example`을 복사해 `.env.local`로 만들고 API 키를 입력합니다:
```bash
cp .env.example .env.local
# .env.local 에서 ANTHROPIC_API_KEY= 에 실제 키 입력
```

**ANTHROPIC_API_KEY가 없어도 Mock 모드로 전체 UI 흐름이 동작합니다** (실제 Mock 구현은 FEAT-05).  
키를 넣으면 실제 `claude-opus-4-8` 모델을 호출합니다.

### FEAT-02 — 로컬 저장 계층 IndexedDB (이슈 #2)

**추가 문서 읽기 이유 기록**: FEAT-01에서 만든 `lib/petContext.tsx` 기존 구조 확인 목적으로 lib/ 폴더 파일 목록을 1회 확인. 타입 정의 중복 방지를 위한 최소 탐색.

**구현 완료 파일 목록**:
- `lib/types.ts` — 도메인 타입 신규 생성
  - 유니온: Species / Gender / Appetite / Activity / Sleep / Toilet
  - 인터페이스: Pet / AISummary / DiaryEntry / AIConversation / MetaRecord
  - Blob 필드(profile_image, photos)에 AI 전송 금지 주석 명시 (보정 지시 4)
  - date: YYYY-MM-DD, created_at: ISO 문자열 타입 주석 (보정 지시 7)
- `lib/db.ts` — IndexedDB 초기화 신규 생성
  - PawprintDiaryDB DBSchema 타입 (idb v8)
  - 4개 스토어: pets(keyPath: pet_id) / entries(keyPath: entry_id) / conversations(keyPath: conversation_id) / meta(keyPath: key)
  - 인덱스: entries.by-pet(pet_id) / entries.by-pet-date([pet_id, date]) / conversations.by-pet(pet_id)
  - 싱글톤 Promise 캐시 + 실패 시 캐시 초기화 (재시도 가능)
- `lib/repos.ts` — Repository 함수 신규 생성
  - metaRepo: get / set
  - petRepo: create / update / get / getActive (metaRepo.activePetId 연계)
  - entryRepo: save / update / get / remove / listByPet / recentByPet
  - conversationRepo: add / listByPet
  - generateId(): crypto.randomUUID + 타임스탬프 폴백
  - sortEntriesDesc(): date desc → created_at desc (보정 지시 7)

**빌드 우려 지점**:
1. idb v8 복합 인덱스 타입 `'by-pet-date': [string, string]` — `createIndex('by-pet-date', ['pet_id', 'date'])` 호출 시 idb가 `keyPath: string | Iterable<string>`을 수용하므로 런타임 문제 없음. 단, idb v8 내부 TypeScript 타입 추론이 복합 인덱스 keyPath에 대해 까다로울 수 있음 → `npm run build` 결과 확인 필요.
2. `db.createObjectStore('entries', { keyPath: 'entry_id' })` 반환값 타입 — idb v8 upgrade 콜백 내 타입 추론. 문제 발생 시 `as IDBObjectStore` 캐스팅 또는 별도 트랜잭션 활용 필요.

**빌드 결과**: Bash 차단으로 직접 실행 불가. 공방장이 아래 명령으로 검증 필요:
```bash

```
